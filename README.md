# PCR-Writer

## EMS ePCR Writing Practice Application

A web-based practice application designed to help Emergency Medical Services (EMS) personnel improve their electronic Patient Care Report (ePCR) documentation skills.

### Features

- **Interactive Practice Form**: Complete ePCR form with all essential fields including:
  - Patient information
  - Incident details
  - Chief complaint
  - History of present illness (HPI) using OPQRST
  - Past medical history using SAMPLE
  - Vital signs
  - Physical assessment
  - Treatment and interventions
  - Patient care narrative
  - Disposition

- **Practice Scenarios**: Pre-loaded realistic scenarios including:
  - Cardiac emergency (chest pain)
  - Trauma (fall with injury)
  - Respiratory distress
  - Diabetic emergency

- **Documentation Tips**: Comprehensive guidance on:
  - General documentation principles
  - Essential mnemonics (CHART, OPQRST, SAMPLE, DCAP-BTLS)
  - Common mistakes to avoid
  - Legal considerations

- **Additional Features**:
  - Save draft functionality (uses browser localStorage)
  - Review PCR before submission
  - Print-friendly format
  - Responsive design for mobile and desktop
  - Professional, user-friendly interface

### Getting Started

1. **Open the Application**
   - Simply open `index.html` in any modern web browser
   - No installation or server required

2. **Practice Mode**
   - Fill out the ePCR form with patient information
   - Use the hints provided for each section
   - Click "Save Draft" to save your progress
   - Click "Review PCR" to see a formatted view of your report

3. **Use Practice Scenarios**
   - Click on the "Practice Scenarios" tab
   - Select a scenario to auto-populate the form
   - Complete any missing information
   - Review the documentation for learning purposes

4. **Review Documentation Tips**
   - Click on the "Documentation Tips" tab
   - Study the guidelines and mnemonics
   - Apply these principles while practicing

### Documentation Mnemonics

**CHART** (Narrative Structure)
- **C**hief complaint
- **H**istory
- **A**ssessment
- **R**x/Treatment
- **T**ransport

**OPQRST** (Pain/Symptom Assessment)
- **O**nset
- **P**rovocation/Palliation
- **Q**uality
- **R**adiation
- **S**everity
- **T**ime/Duration

**SAMPLE** (Medical History)
- **S**igns/Symptoms
- **A**llergies
- **M**edications
- **P**ast medical history
- **L**ast oral intake
- **E**vents leading to emergency

### Best Practices

1. **Be Objective**: Document facts, not assumptions
2. **Be Accurate**: Record exact times and measurements
3. **Be Complete**: Document all relevant findings
4. **Be Timely**: Complete documentation as soon as possible
5. **Be Professional**: Use proper terminology and maintain patient dignity

### Browser Compatibility

- Chrome/Edge (recommended)
- Firefox
- Safari

### Privacy Note

All data entered in this practice application is stored locally in your browser using localStorage. No information is transmitted to any server. This is strictly a practice tool and should not be used for actual patient care documentation.
